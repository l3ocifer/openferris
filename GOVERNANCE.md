# Governance

## Project Model

OpenFerris uses a maintainer-led model.

## Roles

1. **Maintainers**
   - Set roadmap priorities.
   - Review and merge PRs.
   - Own release management and security responses.
2. **Contributors**
   - Propose and implement changes via issues/PRs.
   - Participate in design and review discussions.

## Decision Making

1. Small changes: maintainer consensus in PR reviews.
2. Significant changes: open design issue + documented decision in `docs/`.
3. In case of disagreement: maintainer final decision with rationale.

## RFC Trigger

Open an RFC-style issue before implementation if a change affects:
- public API contracts,
- data model/schema,
- security boundary,
- economic/accounting logic.

## Moderation and Community Health

The project enforces `CODE_OF_CONDUCT.md`.
Maintainers may moderate discussions, lock abusive threads, and ban accounts that repeatedly violate community standards.
